# Tim's Datenpflege - Anleitung für OpenClaw Bot

## Übersicht

Du pflegst 3 JSON-Dateien, die von der Streamlit-App verwendet werden:

1. `data/files.json` - Liste aller Dateien (wird von OneDrive geladen)
2. `data/folder_structure.json` - Ordnerhierarchie aus OneDrive
3. `data/file_summaries.json` - KI-generierte Zusammenfassungen der Dateien

## 1. files.json

**Format:** Array von File-Objekten

```json
[
  {
    "id": "D0B75638021B19A6!sf63cdf711a154b4e9718165b3f25ecb9",
    "name": "1987-01-21_Freudenberg 1119.pdf",
    "date": "2025-12-18",
    "suggested": "Dokumente",
    "index": 23
  }
]
```

**Felder:**
- `id` - OneDrive File ID (eindeutig)
- `name` - Original-Dateiname
- `date` - Datum der letzten Änderung
- `suggested` - Kategorisierungs-Vorschlag
- `index` - Laufende Nummer für Anzeige

**Pflege:** Periodisch aus OneDrive `Dokumente/ScanSnap` laden und aktualisieren.

---

## 2. folder_structure.json

**Format:** Rekursive Ordnerstruktur

```json
{
  "_comment": "Auto-generated by Tim",
  "last_updated": "2026-04-03T13:40:45",
  "root_path": "Dokumente/ScanSnap",
  "folders": {
    "Versicherungen": {
      "subfolders": {
        "Auto ADAC": {
          "subfolders": {}
        },
        "Rechtsschutz": {
          "subfolders": {
            "Provinzial (gekündigt)": {
              "subfolders": {}
            }
          }
        }
      }
    }
  }
}
```

**Regeln:**
- Jedes Ordner-Objekt hat optional `subfolders: {}`
- Leeres `subfolders` = keine weiteren Unterordner
- Beliebige Tiefe möglich (Versicherungen/Rechtsschutz/Provinzial)

**Pflege:** Bei Ordner-Änderungen in OneDrive aktualisieren.

---

## 3. file_summaries.json ⭐ NEU

**Format:** File-ID → Summary Mapping

```json
{
  "_comment": "Auto-generated by Tim - file summaries from OCR/content analysis",
  "last_updated": "2026-04-03T15:00:00",
  "summaries": {
    "D0B75638021B19A6!sf63cdf711a154b4e9718165b3f25ecb9": {
      "summary": "Bausparvertrag mit LBS, Abschlussdatum März 2018, 50.000€ Vertragssumme",
      "keywords": ["Bauspar", "LBS", "Vertrag", "2018", "50000"],
      "page_count": 5,
      "ocr_text": "Bausparurkunde Nr. 12345...",
      "suggested_filename": "2018-03-23_Bausparvertrag_LBS_Tobi_50000.pdf"
    }
  }
}
```

**Felder:**
- `summary` - **Kurze Zusammenfassung** (1-2 Sätze, max. 150 Zeichen)
- `keywords` - **Wichtige Begriffe** für Suche/Filter (5-10 Stück)
- `page_count` - **Seitenanzahl** des Dokuments
- `ocr_text` - **Erster Absatz oder Auszug** (max. 500 Zeichen, optional)
- `suggested_filename` - **Vorgeschlagener neuer Dateiname** (optional, KI-generiert)

**Generierung:**
1. OCR auf PDF durchführen (z.B. mit PyMuPDF, Tesseract, oder Azure AI Vision)
2. KI-Modell (OpenAI GPT, Claude, etc.) fragt:
   - "Zusammenfasse dieses Dokument in 1-2 Sätzen"
   - "Extrahiere 5-10 wichtige Keywords"
   - "Wie viele Seiten hat das Dokument?"
3. Ergebnis in JSON schreiben

**Performance-Tipp:**
- Priorisiere unbehandelte/neue Dateien
- Cache bereits verarbeitete IDs
- Verarbeite im Hintergrund (Queue-System)

## Workflow

```
1. OneDrive pollen (alle X Minuten)
   ↓
2. Neue Dateien gefunden?
   ├── Ja → OCR + KI-Zusammenfassung
   │         → file_summaries.json ergänzen
   │         → Thumbnail generieren
   │         → Git commit & push
   ↓
3. Neue Ordner gefunden?
   ├── Ja → folder_structure.json aktualisieren
   │         → Git commit & push
   ↓
4. files.json aktualisieren (Index, Datum)
   → Git commit & push
```

---

## Git Commands

```bash
# Nach jeder Änderung:
git add data/folder_structure.json data/file_summaries.json data/files.json
git commit -m "Update: 3 neue Zusammenfassungen, 1 neuer Ordner"
git push origin main
```

## UI-Anzeige

In der Streamlit-App wird deine Arbeit angezeigt als:

- **📁 X Ordner geladen** (aus folder_structure.json)
- **📝 Y Zusammenfassungen geladen** (aus file_summaries.json)
- **🕐 Stand: 2026-04-03** (last_updated)

Jede Datei zeigt:
- Thumbnail (von dir generiert)
- 📄 **Zusammenfassung** (direkt neben Thumbnail: summary, keywords, page_count)
- ✏️ **Dateiname** (editierbar, mit `suggested_filename` als Vorschlag)
- Ordner-Auswahl (basierend auf folder_structure.json)

**Neu:** Dateiname-Änderungen werden mitgespeichert (Feld `new_file_name` in decisions.json)

---

## Fehlerbehandlung

- **File ID nicht gefunden** → Summary wird einfach nicht angezeigt (graceful degradation)
- **JSON kaputt** → App nutzt Default-Werte (leere Strukturen)
- **Git Konflikt** → Pull vor Push, dann merge
